# Instrucciones para Claude - Portfolio Project

## Stack Tecnológico

Este proyecto utiliza las siguientes tecnologías. **SIEMPRE** respeta este stack cuando hagas cambios:

### Frontend Framework
- **Next.js 15/16 (App Router)**
  - Estándar de la industria para portafolios profesionales
  - Server-Side Rendering (SSR)
  - Static Site Generation (SSG)
  - React Server Components (RSC)
  - Optimización automática de rendimiento
  - SEO optimizado

### Styling
- **TailwindCSS**
  - Framework CSS utility-first
  - Diseño rápido, limpio y visual
  - Configuración en `tailwind.config.ts`

### Animaciones
- **Framer Motion**
  - Animaciones suaves y fluidas
  - Transiciones profesionales
  - NO usar otras librerías de animación

### Componentes UI
- **Shadcn/ui**
  - Componentes pre-construidos elegantes
  - Basados en Radix UI
  - Totalmente customizables
  - Ubicados en `components/ui/`

### Iconos
- **Lucide React** - Iconos principales del UI
- **React Icons** - Iconos de marcas y sociales (FaGithub, etc.)

## Reglas de Desarrollo

### 1. Estructura de Archivos
- Componentes de secciones en `components/sections/`
- Componentes UI reutilizables en `components/ui/`
- Archivos estáticos (PDFs, imágenes) en `public/`

### 2. Componentes
- SIEMPRE usar `"use client"` cuando se usen hooks o interactividad
- Preferir Server Components cuando sea posible
- Usar TypeScript para type safety

### 3. Styling
- SOLO usar Tailwind CSS para estilos
- NO crear archivos CSS personalizados
- Usar las clases de Shadcn/ui cuando estén disponibles

### 4. Animaciones
- Usar `framer-motion` para todas las animaciones
- Patrón común: `initial`, `whileInView`, `transition`, `viewport`
- Mantener consistencia en delays y duraciones

### 5. Iconos
- Usar Lucide React (`lucide-react`) para iconos de UI
- Usar React Icons (`react-icons`) solo para logos de marcas
- NO instalar otras librerías de iconos

### 6. Buenas Prácticas
- Mantener componentes limpios y legibles
- Reutilizar componentes cuando sea posible
- Seguir convenciones de Next.js App Router
- Optimizar para SEO y performance
- NO crear documentación innecesaria

## Ejemplos de Código

### Estructura de Componente Típica
```tsx
"use client";

import { motion } from "framer-motion";
import { Button } from "@/components/ui/button";
import { IconName } from "lucide-react";

export default function Section() {
  return (
    <section id="section-name" className="container mx-auto px-4 py-16">
      <motion.div
        initial={{ opacity: 0, y: 20 }}
        whileInView={{ opacity: 1, y: 0 }}
        transition={{ duration: 0.5 }}
        viewport={{ once: true }}
      >
        {/* Contenido */}
      </motion.div>
    </section>
  );
}
```

### Uso de Shadcn Components
```tsx
import {
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
} from "@/components/ui/card";
import { Button } from "@/components/ui/button";
```
